    HTTP缓存

    toovk 12/24/2021 http

    可缓存的静态资源：JS、CSS、Img

    # 1）强制缓存

    # 返回头 Cache-Control：

    • 在 Responnst Headers 中

    • 控制强制缓存的逻辑

    • 例如 Cache-Control：max-age=31536000（单位是秒）相当于一年

    # cache-control 的值

    • max-age：设置过期时间

    • no-cache：不用本地缓存，正常向服务端请求，可以用到协商缓存

    • no-store：不用本地缓存，也不用服务端的服务措施

    • private： 只允许最终用户进行缓存

    • public： 允许路由或者代理进行缓存

    补充：关于Expires

    同在 Response Headers 中

    同为控制缓存过期

    已被 Cache-control 代替

    # 2）协商缓存（对比缓存）

    • 它是一个服务器端缓存策略

    • 服务器判断客户端资源，是否和服务器端资源一样

    • 一致的话返回 304 ，否则返回 200 和最新的资源

    什么是资源标识？

    # 在 Response Headers 中，有两种

    # Last-Modified 资源的最后修改时间

    如果说请求带的 If-Modified-Since 和服务端判断一致，表示没有修改过资源，返回304资源。如果修改过，重新返回一个新的Last-Modified和资源

    # Etag 资源的唯一标识（一个字符串，类似人类的指纹）

    如果说请求带的 If-None-Match 和服务端判断一致，返回304资源。如果不一致，重新返回一个新的E-tag和资源

    Last-Modified 和 Etg

    • 会优先使用 Etag

    • Last-Modified 只能精确到秒级

    • 然后资源被重复生成，而内容不变，则 Etag 更精确

    # 3）总结

    首先发送http请求：

    => Max-age判断缓存是否过期

    => 如果没有过期，读取缓存（强制缓存），页面呈现

    => 过期了，判断是否带有 Etag 和 Last-Modified

    => 如果没有带这两个标识，即向服务器发送http请求，再重新返回请求资源和状态码，最后页面呈现

    => 带了这两个标识，向服务器发起 http 请求带 If-None-Match、If-Modified-Since字段

    => 服务器判断是否可用，可用则返回304状态码读取缓存。不可用返回200状态码重新请求资源。
