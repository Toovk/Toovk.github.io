    HTTP状态码以及Headers

    toovk 12/24/2021 http

    # 1）状态码分类

    1xx 服务器收到请求

    2xx 请求成功，如200

    3xx 重定向，如302

    4xx 客户端错误，如404

    5xx 服务端错误，如500

    # 2）常见状态码

    • 200 表示成功

    • 301 永久重定向（配合location，浏览器自动处理），比如浏览器换域名了，返回301状态码和新的域名地址，下次访问自动跳转新域名

    • 302 临时重定向（配合location，浏览器自动处理），下次访问的话还是会访问老地址，等返回302或者其它状态码在进行下一步

    • 304 资源未被修改，请求js、图片、css等资源返回200状态码之后会存进缓存，下次请求没有过期返回304

    • 404 资源找不到

    • 403 没有权限

    • 500 服务器错误

    • 504 网关超时

    # 3）Restful API

    • 一种新的 API 设计方法

    • 传统 API 设计： 把每个url当作一个功能

    • Restful API设计： 把每个url当作一个唯一的资源

    如何设计成一个资源？

    传统 API 设计：

    get请求：/api/create-blog?id=100

    post请求：/api/update-blog?id=100

    Restful API 设计：

    get请求：/api/blog/100

    post请求：/api/blog/100

    put请求：/api/blog/100

    # 4）Http Headers

    Request Headers

    • Accept: 浏览器可接受的数据格式

    • Accept-Encoding: 浏览器可接收的压缩算法，如gzip

    • Accept-language: 浏览器可接受的语言，如zh-CN

    • Connection: keep-alive 一次TCP 连接重复使用

    • cookie: 同域请求都会带上

    • host: 请求的域名是什么

    • User-Agent: 简称（UA）浏览器、系统信息

    • Content-type: 发送数据的格式，如application/json

    • If-None-Match、If-Modified-Since: 缓存请求标识

    Response Headers

    • Content-type: 返回数据的格式，如application/json

    • Content-length: 返回数据的大小，多少字节

    • Content-Encoding: 返回数据的压缩算法，如gzip

    • Set-cookie: 改cookie

    • Cache-Control: Max-age=...

    • Last-Modified、E-tag: 缓存返回标识

    自定义 Header

    headers: { 'X-Requested-With': 'XMLHttpRequest' }，随便定义比如token
